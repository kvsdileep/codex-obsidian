# How to Build Your Own LLM Knowledge Base

## Summary

This clipped article argues that most AI usage starts from zero each session, and that a better pattern is to let an AI maintain a persistent personal wiki. The recommended architecture is simple: preserve raw sources in one folder, let the AI synthesize linked wiki notes in another folder, and query the wiki rather than rebuilding context each time.

## Key Claims

- AI should act less like a one-off answer machine and more like a librarian maintaining an external knowledge base.
- The useful minimum architecture is a raw folder plus a wiki folder.
- Raw sources should stay untouched while the AI writes summaries, concept notes, and indexes.
- Obsidian is a good fit because a vault is just a local folder of markdown files.
